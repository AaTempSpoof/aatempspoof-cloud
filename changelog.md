# AaTempSpoof v14.4 更新公告

## 云端授权与更新

- 捐赠名单、黑名单及模块更新入口由 Surge 迁移至 GitHub，增加 GitHub Pages 与 GitHub Raw 双入口回退，降低单一入口异常导致的校验和更新失败。
- 捐赠名单与黑名单升级为数字签名校验，安装阶段和模块运行阶段都会先验证云端数据完整性，再匹配当前设备 ID。
- 增加受限的离线授权缓存。开机网络暂不可用时可使用有效的签名缓存；首次无缓存时会短暂启动并自动复查，避免因开机网络尚未恢复而长期无法启动。
- 授权失效时不再自动执行卸载脚本，改为停止模块全部功能，并将模块说明显示为“当前设备未处于捐赠名单”。后续授权恢复后会自动还原原说明并重新启动功能。
- 精简主模块云端通信，移除旧版设备心跳、远程补丁等待及日志上传链路，仅保留捐赠授权和版本更新所需请求。

## 温度伪装与热控

- 将开机早期的热控配置覆盖逻辑合并到主二进制，`post-fs-data.sh` 仅负责在正确时机启动对应入口，减少早期脚本体积和重复逻辑。
- 热控覆盖文件改为在 `pid/overlay` 中按需生成并记录挂载，不再为了兼容分区创建无内容的 `system` 目录。
- 「热控配置覆盖」现在统一控制热控配置空挂载、配置改写、热控服务处理、充电限制处理等完整流程；关闭后会清理覆盖并恢复为仅温度伪装模式。
- 扩展 `system`、`vendor`、`odm`、`product` 等分区的热控、温控亮度、游戏温控和充电限制配置识别，提高不同厂商系统的覆盖兼容性。
- 重点优化骁龙 8 Gen 3 及后续骁龙平台的温控配置识别，以及 CPU、GPU、DDR、NPU、LLCC、电池和外围温度伪装节点匹配，提高新平台上的伪装覆盖率。
- 修复充电状态下关闭温度墙后 `/proc/shell-temp` 仍保持伪装的问题，并同步处理 `horae` 的停止与恢复，避免真实温度重新被旧值覆盖。
- 优化热控开关切换、总开关关闭和进程退出时的恢复顺序，减少残留挂载、热控服务未恢复或充电限制状态未复原的问题。

## 电池与充电

- 优化电池 `uevent`、BatteryService 与 `dumpsys battery` 的同步方式，提高电池温度、电量、充电状态和插入状态的刷新一致性。
- 增加真实电量有效值保护，遇到节点短暂返回异常值时优先保留最近一次有效电量，降低状态栏电量短暂跳变或不同步的概率。
- 优化充电期间的状态刷新频率，减少曲线伪装、循环伪装及充电状态切换后的生效延迟。
- 增加循环次数挂载状态检查，挂载失效时会自动重新建立并刷新电池状态。
- 总开关关闭、模块停止或卸载时会主动恢复电池挂载、充电强制值、相关系统属性及热控服务，减少关闭后仍保留伪装状态的问题。

## WebUI 更新

- 温度伪装和温度墙的小数数值现在可以直接点击输入，同时保留原有滑块调节方式，并自动限制到有效范围和步进值。
- 修复「极致伪装」卡片开关光圈被边界裁切及开关位置异常的问题。
- 「极致伪装」现在只以「其他伪装」分类开关作为启用条件；未开启「其他伪装」时无法启用，并修复已开启后仍错误提示“请开启其他伪装”的状态同步问题。
- 更新检查增加 GitHub 固定入口回退，发现旧 Surge 或 Netlify 地址时会自动改用新入口。
- 手动检查更新改为获取最新数据后再显示结果，并延长网络等待时间，减少旧缓存和慢网络造成的误报。
- 自定义模块名称改用纯文本方式显示，避免名称内容被当作 HTML 解析。

## 安装与配套工具

- 修复刷入脚本音量键识别异常导致未选择便继续安装或错误提示“用户取消安装”的问题。
- 安装阶段改为调用主二进制验证签名授权，不再由 shell 脚本直接下载并读取明文名单。
- 检测到 `/data/adb/ksu` 时会自动跳过 KsuWebUI 安装询问。
- 去除 mountify 等元模块依赖，温度节点、热控配置、电池和充电状态所需的 bind mount 均由模块自身完成，无需额外安装元模块即可使用完整功能。
- 优化模块权限设置，目录和普通文件不再统一使用 `0777`，仅对需要执行的脚本和二进制设置执行权限。
- 模块包内置 v14.4 快捷开关 APK，并改为非调试的 release 构建；安装时不再依赖开发目录中的备用 APK。
- `action.sh` 查询页面增加 30 秒无操作自动退出，避免长期占用刷入工具或模块管理器界面。
- 服务启动不再清空现有日志，并继续保留超大日志裁剪，方便排查启动和运行问题。

## 卸载与恢复

- 重写卸载清理流程，增强主进程及内置守护进程的停止和兜底强制结束处理。
- 卸载时会根据挂载记录和 `/proc/self/mountinfo` 清理模块产生的覆盖挂载、电池节点挂载及热控配置挂载。
- 增加 `pid`、overlay、安装校验临时文件、音量键临时文件和相关 cgroup 状态清理。
- 扩展热控、充电、显示刷新率及厂商节点的恢复范围，并重新启动必要的热控和充电服务。
- 主二进制检测到模块管理器的 `remove` 标记时不再重复执行 `uninstall.sh`，避免与模块管理器自身卸载流程互相冲突。
- 卸载时同步移除 AaTempSpoof 快捷开关 APK，减少模块卸载后的配套组件残留。

## 安全与打包

- 加强主二进制的字符串保护、控制流混淆、载荷加密、完整性校验及运行环境防护，提高静态分析和直接修改校验逻辑的成本。
- 增加关键脚本和加密库的内容与权限校验，并加强 `module.prop` 关键字段完整性校验；检测到异常时会触发保护处理，避免在关键文件被替换后继续正常运行。
- 强化打包版 `customize.sh` 的加密执行封装；源码目录仍保留可维护的明文版本，只有最终模块包使用受保护版本。
- 构建脚本现在会自动编译主二进制、加密 `customize.sh`、检查敏感字符串、校验 ZIP 根目录结构并拒绝包含源码的错误包。
- 快捷开关 APK、主二进制、加密 WebUI 和安装脚本统一由发布流程写入最终模块包，降低手工替换遗漏或打入旧文件的概率。

## 升级说明

- v14.4 保留 v14.3 的 4.19、5.4、5.10 及以上内核兼容逻辑，可直接覆盖刷入升级。
- 高级模式会继续读取并迁移已有配置；极简模式仍会移除 WebUI。
- 开机时如网络尚未可用，模块会按授权缓存和自动复查策略处理，不需要反复重启或重新刷入。
- 如授权暂时失效，模块只会暂停功能，不会删除模块配置；授权恢复后可自动继续运行。




# AaTempSpoof v14.3 更新公告

## 兼容性更新

- 新增 4.19 和 5.4 内核刷入支持，保留原有 5.10 及以上内核支持。
- 温度节点检测扩展为 `emul_temp` / `temp` 双路径，提升无 `emul_temp` 设备的温度伪装兼容性。
- 扩展 CPU、GPU、内存、电池、无线充、外围温度等节点识别，提高 4.19 / 5.4 及部分平台的温度识别率。

## 温度伪装与热控更新

- 新增「热控配置覆盖」开关，默认开启，保持旧版本完整热控覆盖逻辑。
- 关闭「热控配置覆盖」后进入仅温度伪装模式，不再执行温控配置空挂载、热控服务处理、`cooling_device` 清理、充电 boost 等完整去温控动作。
- 系统温度墙修改继续作为独立开关控制，不和仅温度伪装模式冲突。
- 优化无 `emul_temp` 节点设备的温度伪装写入方式，提高 CPU、GPU、内存等温度显示生效概率。

## WebUI 更新

- 新增「热控配置覆盖」卡片，可直接在 WebUI 中切换完整热控覆盖或仅温度伪装。
- 捐赠欢迎弹窗增加加载圆圈，检测期间会阻止误触，确认后恢复页面操作。
- 触感反馈震动强度上限提升到 300%。
- 温度显示区域扩展节点匹配，提升 CPU、GPU、内存等分类温度显示兼容性。

## 注意事项

- 默认开启「热控配置覆盖」，刷入后行为与旧版本完整模式一致。
- 如只想保留温度伪装，可在 WebUI 中关闭「热控配置覆盖」。
- 关闭「热控配置覆盖」后，其他独立开关仍按各自设置生效。
