# AaTempSpoof v14.4 更新公告

## 云端授权与更新

- 捐赠名单、黑名单及模块更新入口由 Surge 迁移至 GitHub，并提供 GitHub Pages 与 GitHub Raw 双入口回退，降低单一入口异常造成的校验失败。
- 捐赠名单与黑名单升级为数字签名校验，安装和运行时均会先验证云端数据完整性，再匹配当前设备 ID。
- 增加受限的离线授权缓存；开机网络暂不可用时可使用有效缓存，无缓存时会短暂启动并自动复查，避免长期无法启动。
- 授权失效时不再自动卸载模块，改为停止全部功能并修改模块说明；授权恢复后会自动还原说明并重新启动。
- 移除旧版设备心跳、远程补丁等待及日志上传链路，仅保留捐赠授权和版本更新所需请求。

## 温控与兼容性

- 将开机早期热控配置覆盖逻辑合并到主二进制，`post-fs-data.sh` 仅负责在正确时机启动对应入口。
- 去除 mountify 等元模块依赖，温度节点、热控配置、电池和充电状态所需挂载均由模块自身完成，无需额外元模块即可使用完整功能。
- 重点优化骁龙 8 Gen 3 及后续骁龙平台的温控配置，以及 CPU、GPU、DDR、NPU、LLCC、电池和外围温度伪装节点识别。
- 扩展 `system`、`vendor`、`odm`、`product` 等分区的热控、温控亮度、游戏温控和充电限制配置兼容性。
- 「热控配置覆盖」现在统一控制配置挂载、热控服务和充电限制处理；关闭后会清理覆盖并恢复为仅温度伪装模式。
- 修复充电时关闭温度墙后 `/proc/shell-temp` 仍保持伪装的问题，并同步处理 `horae` 的停止与恢复。
- 优化热控切换、总开关关闭和进程退出时的恢复顺序，减少残留挂载及热控服务未恢复的问题。

## 电池与充电

- 优化电池 `uevent`、BatteryService 与 `dumpsys battery` 的同步，提高温度、电量、充电状态和插入状态的刷新一致性。
- 增加真实电量有效值保护，降低节点短暂返回异常值时状态栏电量跳变或不同步的概率。
- 优化充电期间的状态刷新频率，并增加循环次数挂载检查，失效时会自动重新建立。
- 总开关关闭、模块停止或卸载时会恢复电池挂载、充电强制值、系统属性及相关服务。

## WebUI 更新

- 温度伪装和温度墙的小数数值支持直接点击输入，同时保留滑块调节，并自动限制有效范围和步进值。
- 修复「极致伪装」开关光圈裁切和位置异常问题。
- 「极致伪装」现在只以「其他伪装」开关作为启用条件，并修复已开启后仍错误提示的问题。
- 更新检查增加 GitHub 固定入口回退，旧 Surge 或 Netlify 地址会自动切换到新入口，并减少旧缓存造成的误报。

## 安装与配套工具

- 修复刷入脚本音量键误判导致未选择便继续安装或错误提示“用户取消安装”的问题。
- 安装阶段改为调用主二进制验证签名授权，不再由 shell 脚本直接读取明文名单。
- 检测到 `/data/adb/ksu` 时自动跳过 KsuWebUI 安装询问，并移除 mountify 的选择和跳转逻辑。
- 优化模块权限设置，避免目录和普通文件统一使用 `0777`，同时不再创建无内容的 `system` 目录。
- 模块包内置 v14.4 快捷开关 APK，并改为非调试的 release 构建，不再依赖开发目录中的备用 APK。
- `action.sh` 增加 30 秒无操作自动退出；服务启动不再清空现有日志，方便排查运行问题。

## 卸载与稳定性

- 重写卸载清理流程，增强主进程和内置守护进程的停止、兜底强制结束及状态恢复。
- 卸载时会根据挂载记录清理 overlay、电池节点和热控配置挂载，并清除相关 `pid` 与临时文件。
- 扩展热控、充电、显示刷新率及厂商节点的恢复范围，并重新启动必要服务。
- 主二进制检测到模块管理器的 `remove` 标记时不再重复执行 `uninstall.sh`，避免卸载流程冲突。
- 卸载时同步移除 AaTempSpoof 快捷开关 APK，减少配套组件残留。

## 升级说明

- v14.4 保留 v14.3 对 4.19、5.4、5.10 及以上内核的兼容逻辑，可直接覆盖刷入升级。
- 高级模式会继续读取并迁移已有配置；极简模式仍会移除 WebUI。
- 开机网络暂不可用时会按授权缓存和自动复查策略处理，无需反复重启或重新刷入。
