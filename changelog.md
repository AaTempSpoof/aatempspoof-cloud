# AaTempSpoof v14.5 更新公告

以下内容以 v14.4 正式源码为基准，仅包含 AaTempSpoof 模块本体的更新与变化。

## 新增内核电池温度伪装

- 在附加页面的「温度墙」下方新增独立的「内核电池温度伪装」卡片，不与原有电池温度伪装、温度墙共用开关。
- 新增上层、底层两级接管，并允许分别开启：上层接管欧加 `gauge_dbg_tbat` 充电温度，底层接管 `virtual_gauge`、`adsp_gauge`、排线座 BTB 与 `usb-therm`。
- 底层会自动扫描 `thermal_zone` 与 IIO 温度通道，适配不同机型上 `usb-therm` 节点编号、路径不一致的问题，并在运行中定期重新发现节点。
- 支持 20～45℃固定温度及 10 分钟动态往返温度；上层、底层共用温度值，但接管状态相互独立。
- 伪装启用后不以真实电池温度达到 55℃作为自动停止条件，避免真实温度变化导致内核伪装被提前释放。
- 完善安全复位：关闭对应开关、关闭模块总开关、进程异常退出、模块停用或卸载时，都会清除内核写入并恢复相关节点。

## 电池与充电状态

- 新增「充电时禁用电池伪装」开关。开启后仅在检测到充电时暂停电池温度伪装，并让 `/proc/shell-temp` 保持原有充电逻辑；停止充电后自动恢复伪装。
- 重做 BatteryService 状态释放流程：关闭电池伪装或电量伪装后会主动执行复位并触发电池事件，避免测试状态残留造成电量、充电状态或快充提示不刷新。
- 真实电量 0% 现在会被当作有效电量同步，不再错误回退到上一次电量。
- 电池温度节点从伪装恢复真实值时增加短暂重试，减少切换瞬间显示 0℃的问题。
- 温度墙关闭且不再写入 `/proc/shell-temp` 时，会同步处理 Horae 的接管状态，减少系统重新写回造成的状态冲突。

## 热控覆盖与开机流程

- 热控配置覆盖由刷入时生成改为每次开机早期根据当前系统文件动态生成，ROM 更新或分区文件变化后不再继续使用旧副本。
- 所有热控配置均在模块运行目录内生成，并通过临时 bind mount 生效，不直接写入实际的 system、vendor、odm 等分区。
- 关闭「热控配置覆盖」时会主动卸载并清理本次生成的覆盖文件，避免旧挂载继续残留。
- `post-fs-data.sh` 改为由主二进制统一执行早期挂载与热控处理，减少脚本逻辑分散、旧文件被重复调用的问题。
- 卸载时不再清空 `/data/vendor/thermal/config`、`/data/thermal/config` 等厂商热控配置目录，保留系统原始配置。

## WebUI 更新

- 新增纯黑背景模式，并针对卡片、描边、文字、滑块、开关和弹窗使用分层灰色，降低纯黑界面的高对比度和刺眼感。
- 温度数值除滑块外支持直接输入，CPU、GPU、内存、电池、其他温度及温度墙均可更精确地填写小数。
- 修复「极致伪装」状态读取与保存不一致的问题；刷新页面或重新进入 WebUI 时会读取实际配置，不再被普通页面刷新意外关闭。
- 配置保存改为临时文件写入后原子替换，并对关键写入结果进行检查；保存失败时会明确提示 Root 权限或管理器执行异常。
- 动态温度、附加页面开关及卡片可用状态会在读取配置后统一同步，减少界面显示与实际运行状态不一致。
- 检查更新改为每次手动检查都重新获取远端数据，增加有效地址回退与更合理的等待时间，减少旧缓存导致的版本判断错误。
- 优化开关光圈、滑块数值区、底栏和弹窗在不同主题下的显示细节。

## 安装与升级

- 覆盖刷入前会先备份现有配置，再停止正在运行的旧模块进程；支持从 `AAaTempSpoof`、`AaTempSpoof`、`AaTempdc` 等旧目录迁移用户配置。
- 旧模块改为先安全停止并标记重启后清理，避免升级过程中直接调用旧版卸载脚本或误删新版本文件。
- 内置快捷开关 APK 改为保留用户数据的覆盖升级，不再先卸载旧版；同时修复连续点击造成的重复切换及 Root 命令超时处理。
- KernelSU 环境会自动跳过不需要的 KsuWebUI 安装询问，并移除 v14.4 中的 mountify 安装提示与依赖说明。
- 模块文件权限由全目录 `0777` 调整为目录 `0755`、普通文件 `0644`、可执行文件单独 `0755`，减少不必要的可写权限。
- 模块操作脚本等待音量键时增加 30 秒自动退出，避免查询页面长期卡住。

## 运行、卸载与安全

- 守护进程会同时校验 PID 与实际启动参数，避免 PID 复用时把无关进程误判为模块进程或在卸载时误杀同名进程。
- 移除开机 60 秒后的强制完整重初始化，减少模块已正常运行时被重复停止、重启的情况。
- 卸载流程新增内核电池节点、临时热控挂载、覆盖映射、充电属性和系统属性的完整恢复，并同时清理待安装目录残留。
- 在线捐赠校验升级为签名授权校验，增加有效授权缓存与黑名单校验；短时网络异常时可使用 72 小时内的有效缓存。
- 授权暂时失效时改为暂停模块功能并定期复查；授权恢复后会自动还原模块状态并重新启动功能，无需重新刷入。
- 加强主二进制、安装脚本与 WebUI 的加密、混淆和完整性校验，提高篡改与静态逆向成本。
- 完整性异常时不再自动卸载模块或删除用户配置，而是安全停止功能并提示重新刷入。

## 从 v14.4 升级

- 可直接覆盖刷入 v14.5，安装器会迁移原有伪装、温度墙、动态温度及其他 WebUI 配置。
- 「内核电池温度伪装」和「充电时禁用电池伪装」均为新增功能，升级后默认关闭，不会自动改变 v14.4 的电池与充电行为。
- 原有设备 ID 继续沿用，无需重新注册；完成刷入后请重启一次，让新的开机早期热控覆盖与复位流程完整生效。
